# 🎯 Boss Hunter — BOSS直聘求职助手（浏览器扩展）

一款稳定运行于 **Microsoft Edge** / Chrome 的 BOSS直聘（zhipin.com）求职辅助扩展。支持**自动打招呼、批量沟通、岗位智能过滤、投递去重与统计**，并提供可视化设置面板与页面悬浮控制台。

> 本项目为开源学习项目，参考自同类「boss-hunter」浏览器插件思路实现，基于 **Manifest V3** 编写，纯原生 JavaScript，无需任何构建步骤，开箱即用。

---

## ✨ 功能特性

| 功能 | 说明 |
| --- | --- |
| 🤖 自动打招呼 | 在职位列表页一键向多个岗位的 Boss 自动发送打招呼语 |
| 💬 批量沟通 | 自动遍历当前页岗位卡片，依次点击「立即沟通」并发送消息 |
| 🧠 岗位智能过滤 | 按 **必含关键词 / 排除关键词 / 薪资区间 / 公司规模** 自动跳过不合适岗位 |
| 🔁 投递去重 | 基于「岗位名 + 公司 + 薪资」指纹去重，避免对同一岗位重复打招呼 |
| 📊 投递统计 | 实时统计「今日 / 累计打招呼数」与「跳过数」，跨天自动清零今日计数 |
| ⚙️ 自定义打招呼语 | 支持 `{jobName}`、`{companyName}` 占位符，按岗位动态生成话术 |
| ⏱️ 频控保护 | 可配置随机操作间隔，模拟人工节奏，降低被风控概率 |
| 🪟 悬浮控制台 | 页面内悬浮面板，随时开始/停止并查看实时运行日志 |

## 📦 安装方法（Edge）

本扩展为「解压的扩展程序」，无需打包即可加载：

1. 下载本仓库（`Code → Download ZIP`）并解压，或 `git clone` 到本地。
2. 打开 Edge，地址栏输入并回车：`edge://extensions/`
3. 打开右下角 / 左侧的 **「开发人员模式」**（Developer mode）开关。
4. 点击 **「加载解压缩的扩展」**（Load unpacked）。
5. 选择本项目根目录（即包含 `manifest.json` 的文件夹）。
6. 安装完成后，工具栏会出现 🎯 图标。建议点击「固定」便于使用。

> Chrome 安装方式相同：访问 `chrome://extensions/` → 开启开发者模式 → 加载已解压的扩展程序。

## 🚀 使用步骤

1. 登录 [BOSS直聘](https://www.zhipin.com/) 并进入**职位推荐 / 搜索结果**列表页。
2. 点击工具栏的 🎯 图标，打开设置面板：
   - 填写**打招呼语**（可用占位符）；
   - 按需配置**过滤规则**、**本轮上限**与**操作间隔**；
   - 点击 **「保存设置」**。
3. 两种启动方式任选其一：
   - 在设置面板点击 **「在当前页运行」**；
   - 或使用页面右下角的**悬浮控制台**，点击 **「开始打招呼」**。
4. 运行过程中可在悬浮控制台查看实时日志，随时点击 **「停止」**。

## ⚙️ 配置项说明

| 配置 | 默认值 | 含义 |
| --- | --- | --- |
| 打招呼语 | 预置话术 | 发送给 Boss 的消息，支持 `{jobName}` / `{companyName}` |
| 本轮上限 | 30 | 单次运行最多打招呼数量 |
| 间隔(秒) | 3 ~ 6 | 每次操作之间的随机等待区间 |
| 启用岗位过滤 | 开 | 是否启用下方过滤规则 |
| 必含关键词 | 空 | 岗位信息须命中任一关键词才打招呼，留空不限制 |
| 排除关键词 | 外包,驻场,销售,电话客服 | 命中任一即跳过 |
| 薪资下限 / 上限(K) | 0 / 0 | 0 表示不限制；与岗位薪资区间做交集判断 |
| 公司规模排除 | 空 | 匹配公司标签（如 `0-20人`）则跳过 |

## 🛠️ 技术说明 & 适配

- **Manifest V3**，后台采用 ES Module 形式的 Service Worker。
- 内容脚本对每个关键 DOM 元素都提供**多组候选选择器**（selector fallback），并使用 `MutationObserver` 等待异步渲染，提升对 BOSS直聘改版的容错性。
- 对 React 受控输入框采用原生 `value` setter + `input` 事件派发的方式填值，确保框架能感知到内容变化。
- 若官方大改版导致元素定位失效，只需更新 `content/content.js` 顶部的 `SELECTORS` 常量即可，无需改动其他逻辑。
